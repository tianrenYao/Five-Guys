### 一、高优先级（补全核心流程，提升实用价值）
#### 1. 未完成功能落地（优先闭环核心能力）
| 功能 | 落地建议 | 价值 |
| --- | --- | --- |
| 管理员用户管理 | 开发 `/admin/user-crud` 页面，支持 admin 角色对全量用户的增删改查、角色分配、密码重置 | 补齐 RBAC 体系最后一环，实现用户生命周期管理，满足企业级权限管控需求 |
| CSV 导出 | 为碳排/废弃物记录列表页增加「导出CSV」按钮，支持按日期/门店/类别筛选后导出，字段包含记录ID、门店、类别、数值、时间、备注 | 满足企业数据存档、跨系统同步、财务/审计对账的核心需求 |
| 审计日志查看页 | 完成 `/admin/audit-log` 开发，展示操作人、操作类型（新增/修改/删除）、操作对象（碳排记录/废弃物记录）、操作时间、IP地址 | 满足ESG管理的合规性要求，支持追溯数据变更源头 |
| 邮件通知（预警触发） | 基于现有预警系统，新增「预警通知规则」配置页（hq_manager 可操作），支持按指标/区域/门店绑定收件人邮箱；预警触发时自动发送邮件（用 Flask-Mail 实现） | 提升预警时效性，避免人工巡检遗漏关键异常 |


#### 2. 核心功能体验优化
+ **碳足迹/废弃物记录优化**：
    - 新增「批量导入」功能（Excel/CSV），支持门店批量上传月度数据（解决单条录入效率低的问题）；
    - 增加「记录修改/撤回」功能（按角色控制：门店员工可撤回24h内未审核记录，区域/总部经理可修改全量记录）；
    - 记录列表页增加「数据对比」按钮（如本月vs上月、本门店vs同区域均值），直观展示趋势。
+ **Dashboard 个性化**：
    - 支持用户自定义看板布局（拖拽调整图表位置/大小）；
    - 新增「关键指标预警模块」（看板顶部展示当前未处理预警）；
    - 图表增加「下钻」功能（如点击区域柱状图可展开该区域下所有门店数据）。
+ **PDF报告优化**：
    - 补充 SDG 对齐章节的可视化图表（基于现有 `company_sdg` 表，展示各SDG目标的进度占比）；
    - 报告新增「数据来源说明」「计算方法备注」（符合ESG报告合规要求）；
    - 支持报告模板自定义（hq_manager 可配置报告是否包含AI评论、SDG章节、区域对比等）。

### 二、中优先级（拓展场景，增强数据价值）
#### 1. ESG 场景拓展
+ **能源消耗精细化管理**：
    - 在碳足迹模块新增「能源类型细分」（如电力区分光伏/电网、燃料区分天然气/柴油）；
    - 新增「能源使用效率分析」（单位营收碳排放、单店日均能耗），并在看板展示。
+ **社会维度（S）补充**：
    - 新增「员工可持续发展培训记录」模块（门店/区域经理录入培训人数、时长、内容）；
    - 新增「供应商ESG评估」模块（总部录入供应商的碳排放标准、废弃物回收能力等评分）。
+ **治理维度（G）补充**：
    - 新增「ESG政策管理」页面（展示公司ESG政策文档、更新记录，支持按角色查看）；
    - 关联审计日志，新增「合规性评分」（基于数据完整性、预警处理及时性等维度自动计算）。

#### 2. 技术能力增强
+ **AI 能力落地**：
    - 接入 DeepSeek API 完成「AI报告评论」功能，支持自定义评论维度（如趋势分析、风险提示、优化建议）；
    - 新增「AI数据异常检测」（对录入的碳排/废弃物数据做智能校验，如数值超出同门店历史3倍范围时提示异常）。
+ **OCR 体验优化**：
    - 支持移动端拍照上传账单（适配响应式布局，优化手机端上传体验）；
    - 新增 OCR 识别结果编辑页（自动提取后可手动修正字段，减少录入错误）；
    - 支持更多账单类型（如水费、运输费发票），扩充排放因子库。

### 三、低优先级（长期迭代，提升产品竞争力）
1. **多端适配**：开发移动端H5版本（重点适配看板查看、预警提醒、简单记录录入）；
2. **数据可视化升级**：引入3D可视化（如门店碳排放热力图）、动态趋势动画；
3. **第三方集成**：
    - 对接企业ERP系统（自动同步门店营收、员工数据，用于能耗效率分析）；
    - 对接碳排放交易所API（自动获取最新排放因子、碳价数据）；
4. **智能决策辅助**：基于历史数据训练简单的预测模型，预测下月碳排/废弃物产生量，给出减排建议；
5. **多语言支持**：适配中英文切换（满足跨国企业/外资门店使用需求）。

### 四、落地建议
1. **分阶段推进**：
    - 第一阶段（1-2周）：完成「管理员用户管理」「CSV导出」「审计日志查看页」（补全核心未完成功能）；
    - 第二阶段（2-3周）：优化核心体验（批量导入、Dashboard下钻、PDF报告SDG模块）；
    - 第三阶段（1个月）：落地邮件通知、AI异常检测、能源精细化管理；
2. **先验证再落地**：对「社会/治理维度拓展」等非核心功能，可先基于现有数据库表开发MVP版本（仅展示核心字段），验证用户需求后再完善；
3. **技术适配**：
    - 批量导入/导出可复用现有Flask路由，引入 `pandas` 处理Excel/CSV；
    - 邮件通知基于Flask-Mail，配置SMTP服务器即可快速落地；
    - AI功能可先做Mock版本（返回预设评论/检测结果），再接入真实API。

这些建议既延续了现有平台的「角色权限管控」「数据可视化」「合规性」核心设计思路，又贴合ESG管理的行业趋势，优先解决用户「数据录入效率低」「预警不及时」「报告维度不全」等痛点，同时为长期拓展预留了技术和功能空间。
