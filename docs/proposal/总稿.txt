Part A: ProblemBusinesses, especially small and medium-sized ones, lack a unified, affordable software tool to track carbon emissions, manage waste classification data, and generate compliance-aligned sustainability reports. Currently, most enterprises rely on manual Excel-based record-keeping, which is prone to human error, unable to visualize data trends for actionable insights, and fails to effectively align their sustainability efforts with the UN Sustainable Development Goals (SDGs). Additionally, commercial ESG and sustainability management software is cost-prohibitive for SMEs, leaving them without accessible solutions to monitor, optimize, and report on their environmental impact— a critical need as businesses aim to adopt sustainable operations and meet regulatory and stakeholder demands for transparency.Part B: VisionThis project will deliver a one-stop web-based sustainability management platform that enables businesses to automatically calculate carbon footprints and track waste classification recycling rates through simple, intuitive data entry. The platform will support the automatic generation of monthly, quarterly, and annual sustainability reports with one-click PDF export, streamlining compliance and stakeholder reporting. Hosted on the university-provided virtual machine, the platform will be fully accessible via remote web browsers, allowing assessors in Ireland to test all core functionalities directly and seamlessly using pre-provided test accounts.Part C: BenefitsFirst, the platform drastically reduces manual data statistics time by automating carbon emission calculations through the standard formula of activity value multiplied by emission factor. Second, it enhances data accuracy via MySQL database constraints and built-in standard emission factor libraries, eliminating errors from manual entry and calculation. Third, it integrates ECharts to provide visual data trends through charts (line, pie, bar), making environmental performance insights clear and actionable. Fourth, the system is pre-aligned with UN SDG 9, 11, 12 and 13, enabling businesses to directly map their sustainability efforts to global goals. Fifth, the one-click PDF export function simplifies the creation of formal sustainability reports, supporting enterprises in meeting regulatory compliance and stakeholder reporting requirements efficiently.

# 3. Success Criteria

The following success criteria are defined using the **SMART** framework (Specific, Measurable, Achievable, Relevant, Time-bound) to ensure clear, objective evaluation of project outcomes.

**SC1 — Backend API Completion**
All 22 RESTful API endpoints (covering authentication, carbon tracking, waste management, dashboard analytics, and report generation) shall be fully implemented and pass automated integration testing via Postman by **Week 8**. Each endpoint must return standardised JSON responses with appropriate HTTP status codes.

**SC2 — Frontend Interface Delivery**
Five complete business pages (Login, Dashboard, Carbon Tracking, Waste Management, and Report Generation) shall be developed with responsive layouts (Bootstrap 5) and interactive data visualisations (ECharts 5), rendering correctly on the latest versions of Chrome and Firefox, by **Week 12**.

**SC3 — Remote Deployment and Accessibility**
The platform shall be deployed to the university-provided virtual machine and accessible via a public URL by **Week 14**, enabling the examiner to remotely log in, navigate all modules, and verify functionality from any location using only a standard web browser.

**SC4 — Data Security and Access Control**
The system shall enforce role-based access control with two distinct roles (Business Admin and Staff). All user passwords must be stored using Werkzeug PBKDF2-SHA256 hashing. Every API endpoint (except the login page) shall require session-based authentication, and all database queries shall use parameterised statements to prevent SQL injection.

**SC5 — Sustainability Report Generation and Export**
The report module shall support three reporting periods (monthly, quarterly, annual) and a custom date range. Each generated report must automatically aggregate carbon emission and waste management data for the selected period, and the system shall export reports as downloadable PDF documents containing data summaries, trend analysis, and SDG alignment indicators.

5、Software AttributesFunctionality:Our platform mainly includes three key functions: tracking carbon emissions, managing waste classification, and automatically generating sustainability reports that align with UN goals (SDG 9, 11, 12, 13). Internally, we designed a set of standardized system interfaces (RESTful APIs) so that all these modules can work smoothly together and also support future expansion or integration with other systems.Performance:We ensure the system responds quickly by optimizing how data is stored and searched in the database, and by using efficient chart tools (like ECharts) for data display. In addition, static content is delivered through CDN, which helps reduce loading time for users.Maintainability:The system is built in a modular way, meaning different parts (like frontend and backend) are clearly separated. This makes it easier to update, fix, or add new features in the future without affecting the whole system.Reliability:We use strict data rules in the database to make sure all information stays accurate and consistent. At the same time, the system has unified error handling, so it behaves in a stable and predictable way even when something goes wrong.Security:User data security is a key priority. We apply encrypted password storage, login authentication, and role-based access control to ensure that only authorized users can access certain data. We also use secure query methods to prevent common attacks like SQL injection.Efficiency:To make better use of system resources, we reuse database connections and load common libraries (like Bootstrap and ECharts) through CDN. This reduces server workload and improves overall efficiency.Usability:The platform is designed with a user-friendly interface using responsive layout, so it works well on different devices. Forms are simple to use, and data is presented through clear visual charts. We also provide a demo account so users can quickly experience the system.
Remote Testing (slightly improved):The system is deployed on a virtual machine hosted by University College Dublin and can be accessed remotely via a secure gateway. Users are provided with test accounts and clear instructions, allowing them to log in and test the platform from different locations in a controlled and secure way.
