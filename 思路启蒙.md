## 一、先定核心定位：给瑞幸做的是什么系统？
你面对的客户是**连锁咖啡品牌（瑞幸）**，业务核心是：门店多、饮品多、包装/耗材多、ESG（碳足迹/废弃物/报告）监管与品牌披露压力大。  
所以你们的系统本质是：

> **瑞幸ESG数据采集 + 计算 + 管理 + 报告自动化平台（面向总部/区域/门店三级）**
>

三个功能要落在“**可采集、可计算、可审计、可披露**”这四个字上。

## 二、三个功能分别该记录什么（数据口径先定下来）
下面给你一套**可落地、符合连锁咖啡场景**的数据口径清单（你可以直接当“需求文档第一章”用）。

### 1）碳排放追踪（Carbon Emission Tracking）
#### 1.1 要记录的核心维度（建议分三大块）
**A. 范围1：直接排放（门店/物流/设备）**

+ 门店直接排放：空调/冰箱/咖啡机等用电对应的**排放因子**（按电网排放因子）、燃气/热水等燃料消耗
+ 物流配送：冷链车辆油耗/电耗、行驶里程、配送半径
+ 包装/耗材：采购环节的运输与仓储排放（可选先做重点）

**B. 范围2：外购电力/热力排放**

+ 门店月度用电量（kWh）
+ 区域中心/中央工厂用电（如果你们要算到供应链层级）

**C. 范围3：上游供应链（可选做二期/重点先做范围1+2）**

+ 生豆/牛奶/糖浆/纸杯/杯盖/吸管/包装袋等供应商的碳排放数据（采集难度高，先做“披露口径对标+抽样”）

#### 1.2 每条记录必须有的字段（建表/接口都按这个来）
| 字段 | 说明 | 粒度 |
| --- | --- | --- |
| 门店ID/区域ID/总部ID | 组织层级 | 门店级是核心 |
| 统计周期 | 日/周/月（建议月汇总，日明细） |  |
| 排放范围 | 1/2/3 |  |
| 排放类别 | 用电/燃气/冷链/包装/耗材 |  |
| 活动数据（Activity Data） | 用电量(kWh)、耗气量(m³)、里程(km)、包装用量(个/kg) | 数值 |
| 排放因子（EF） | 如tCO₂e/单位 | 可配置（国标/行业/企业自定义） |
| 排放量 | 自动计算：活动数据 × EF | 自动 |
| 计算方法/版本 | 可追溯（方法学变更） |  |
| 凭证/附件 | 电费单、燃气单、配送单据等（链接/上传） |  |
| 录入人/审核人/时间 | 审计 |  |


> 关键：**排放量尽量自动计算，人工只填“活动数据+上传单据”**，否则后期不可信。 图像识别

### 2）废弃物处理量（Waste Management）
瑞幸的痛点通常是：**包装废弃物、餐厨垃圾、危废（清洁用品/耗材）**。建议分三类记录，口径清晰。

#### 2.1 要记录的核心维度
**A. 包装类废弃物（重点，可披露）**

+ 纸杯/杯盖/吸管/搅拌棒/包装袋/外卖袋等
+ 重点看：**产生量、回收量、回收率、去向**

**B. 餐厨/门店垃圾**

+ 咖啡渣、食物残渣、保洁垃圾等
+ 重点看：产生量、清运方式、处置去向（填埋/焚烧/堆肥）

**C. 危险废物（合规必须）**

+ 清洁剂/化学品容器、废电池/设备等
+ 重点看：类别、产生量、处置单位资质、转移联单（合规台账）

#### 2.2 每条记录建议字段
| 字段 | 说明 |
| --- | --- |
| 废弃物类别 | 包装/餐厨/危废（细项可枚举） |
| 规格/材质 | 纸/塑/铝/可降解/复合材 |
| 产生量 | kg/个/箱（统一单位，建议kg） |
| 回收量/再利用量 | kg |
| 处置方式 | 回收/填埋/焚烧/堆肥/委托处置 |
| 处置单位/资质编号 | 合规 |
| 清运单据编号/日期 | 可追溯 |
| 异常说明 | 超量/缺失/数据异常原因 |


> 建议：**包装废弃物拆成“可回收物+不可回收物”两条明细**，方便算回收率、再利用率。
>

### 3）报告（Reporting & Disclosure）
这里不是“做一张Excel报表”，而是做：**披露级报告 + 内部管理驾驶舱**。

#### 3.1 报告应包含的内容（按客户视角）
**① ESG披露类报告（对外/监管）**

+ 碳排放：范围1+2（范围3可选）、趋势、减排目标进展
+ 废弃物：产生量、分类占比、回收/处置情况、目标达成
+ 水资源/能源（可扩展）

**② 内部管理类报告（门店/区域/总部）**

+ 门店能耗排行（节能降本）
+ 废弃物分类率排行（运营考核）
+ 异常预警（超耗、超量、缺失单据）

**③ 审计与台账**

+ 数据溯源：谁填、谁审核、变更记录、附件链接
+ 计算方法说明、因子版本说明

#### 3.2 报告的“记录”本质是什么？
报告不是单独再记一堆数据，而是：

> 从“碳排放表+废弃物表+门店基础信息表”按维度**汇总/钻取**生成。
>

所以你设计的不是“报告表”，而是**多维模型 + 汇总口径**。

## 三、功能怎么设计：模块与流程（从0到1落地）
### 3.1 整体架构（建议分4层）
1. **数据接入层**：门店填报 + 系统对接（电表/燃气表/ERP/物流系统）
2. **数据治理层**：校验、规则、因子、审计、版本管理
3. **计算层**：自动计算公式、口径规则、分摊规则（区域/门店）
4. **展示层**：驾驶舱 + 报告 + 导出 + 披露

### 3.2 核心流程（给瑞幸的落地路径）
**流程1：碳排放采集与计算**

+ 总部定义：因子库、计算口径、周期（月）
+ 区域/门店：每月填报用电量/耗气量/冷链数据 + 上传单据
+ 系统自动计算排放量（可审核）
+ 自动生成：门店排放、区域汇总、趋势

**流程2：废弃物采集与台账**

+ 门店按类别填报产生量/回收量/处置去向
+ 系统校验：异常值（突然翻倍/为0）预警
+ 生成台账：可导出给合规/审计
+ 计算指标：回收率、再利用率、单位产量排放（可选）

**流程3：报告自动化**

+ 按披露周期（季度/年度）一键生成
+ 支持：多版本（草稿/正式）
+ 支持：导出PDF/PPT/Word（客户要什么给什么）
+ 支持：数据钻取（点一下全国→点一下某省→某门店）

## 四、具体页面怎么设计（直接给你一套前台页面蓝图）
你可以按“角色页面”来拆，更像产品。

### 4.1 角色划分（至少3类）
+ **门店操作员**：填报、上传、查看本门店数据
+ **区域管理员**：汇总、审核、预警处理
+ **总部ESG经理**：全局驾驶舱、报告、配置因子/口径

### 4.2 页面清单（建议10个核心页面）
1. **首页/驾驶舱（总部）**：总览指标 + 趋势 + 预警
2. **碳排放管理**：列表/日历/填报
3. **碳排放详情**：明细、计算链路、单据、审计
4. **废弃物管理**：分类台账、填报、导出
5. **废弃物详情**：处置单位、联单号、异常处理
6. **报告中心**：报告列表、生成、预览、下载
7. **报告配置**：披露模板、章节、口径
8. **因子库/规则**：排放因子、分摊规则、单位映射
9. **门店/区域管理**：组织树、权限、考核
10. **审计日志**：变更记录、导出留痕

### 4.3 给你一个“门店填报页”最实用设计（可直接画原型）
**碳排放填报页（门店）**

+ 顶部：周期选择、保存/提交/撤回、审核状态
+ 左侧：分类tabs（用电/燃气/冷链/包装）
+ 右侧：表格 + 上传区
+ 表格每行列：
    - 月份
    - 用量（输入）
    - 因子（展示/可配置）
    - 排放量（自动）
    - 单据上传（附件）
+ 底部：异常提示（如“用电量为0请确认”）

**废弃物填报页（门店）**

+ 顶部：周期、保存/提交
+ 表格：废弃物类别枚举 + 单位 + 产生量 + 回收量 + 处置方式 + 处置单位 + 上传
+ 汇总栏：本次产生总量、回收总量、回收率（自动）
